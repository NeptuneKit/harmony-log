# 2026-04-01

## 关键决策
- 将 `apple/swift-log` 正式登记为参考项目，作为 HarmonyOS 版本的 API 和行为语义对齐基线。
- 第一批对齐优先项采用：`LoggingSystem.bootstrap` + `LogEvent` 调用点上下文（`source/file/function/line`）+ 基础自动化测试。

## 已完成里程碑
- 新增 `LoggingSystem.bootstrap` 一次性初始化语义。
- `Logger` 支持按 logger label 通过 bootstrap factory 生成专属 handler。
- 日志记录新增调用点上下文字段，并在格式化输出中可见。
- 新增基础单元测试（4 个用例），覆盖 bootstrap 单次约束、label 工厂行为、上下文透传、格式化输出。

## 风险与后续
- 当前项目缺少标准 hvigor 配置文件（`hvigor/hvigor-config.json5`），暂无法执行 HAR 构建命令进行完整构建验证。
- 下一批建议对齐：结构化 MetadataValue、MultiplexLogHandler、NoOp handler、metadata provider/MDC。

## 第二批对齐（已完成）
- metadata 类型升级为结构化：支持标量、数组、字典递归嵌套。
- metadata 合并规则对齐：`base(logger.metadata) + explicit(log-site metadata)`，同 key 以后者覆盖前者。
- 删除语义：当 explicit 或 `setMetadata` 的值为 `undefined` 时，删除该键。
- `formatRecord` 支持结构化 metadata 文本输出（稳定键序）。
- 新增 3 个测试：结构化输出、覆盖/删除合并规则、`setMetadata(undefined)` 删除行为。

## 第三批对齐（已完成）
- 新增 `MultiplexLogHandler`：
  - 初始化 `minimumLevel` 为子 handler 的最小级别。
  - `log(record)` 时按各子 handler 的 `minimumLevel` 分发。
  - 设置 multiplex 的 `minimumLevel` 时同步更新所有子 handler。
- 新增 `HarmonyLogNoOpHandler`：默认 no-op 丢弃所有日志。
- 新增 4 个测试覆盖 multiplex 与 no-op 语义。

## 第四批对齐（已完成）
- 新增 `HarmonyLogMetadataProvider`，支持在日志发出时注入运行时 metadata。
- `LoggingSystem.bootstrap(factory, metadataProvider?)` 支持全局 provider 注入。
- metadata 合并顺序对齐为：`base -> provider -> explicit`。
- 新增 `MDC` 与 `MDCStore`：支持 `set/get/clear` 和 `MDC.metadataProvider()`。
- 新增 3 个测试覆盖 provider 注入、合并顺序、MDC 清理行为。

## 第五批对齐（已完成）
- 新增 `StreamLogHandler`，支持 `standardOutput(label)` / `standardError(label)`。
- 支持 handler 级 metadata 与 metadataProvider。
- `StreamLogHandler` 内部 metadata 合并顺序：`handler metadata -> handler provider -> record metadata`。
- 新增 3 个测试覆盖输出目标与合并语义。

## 第六批对齐（已完成）
- 新增惰性消息/metadata 输入（闭包），低于 logger level 或 compileTimeMinimumLevel 时不求值。
- 新增 `Logger.setCompileTimeMinimumLevel(...)`，用于构建级别日志裁剪（运行时常量模拟）。
- 新增 `source` 回退推断：未传 source 且传了 file 时，自动按路径推断模块名。
- 新增 `Logger.copy()`，支持显式值语义式拷贝。
- 新增 `InMemoryLogHandler`（entries 收集 + clear）。
- 新增 5 个测试覆盖以上能力。
